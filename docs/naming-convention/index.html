Naming Convention

This document refers to the guidelines for naming & declaration rules of variables and class names of a plugin.

  1. Plugin Name: It is the ex: "Auth".
  2. Class Name: The name of the class will be your plugin name suffixed with 'Plugin' i.e. "AuthPlugin".
  3. getPluginName: When returning the name of the plugin, we add <plugin_name> with '@' and followed by the version ex : Auth@1.0.0 (<plugin_name>@).
  4. Adding functions: When adding the function, we can use anything as the name of the function, while <plugin_name>.<function_name> is used conventionally ex : Auth.authenticate
  5. Removing functions: We can remove the function by providing the same name in the removeFunction which we used in the addFunction i.e. <plugin_name>.<function_name

Example :

AuthPlugin.tsx

import React from "react";
import { IPlugin, PluginStore } from "react-pluggable";

class AuthPlugin implements IPlugin {


  getPluginName(): string {
    return "Auth@1.0.0";                                               //line 2
  }

  getDependencies(): string[] {
    return [];
  }

  public pluginStore: any;

  init(pluginStore: PluginStore): void {
    this.pluginStore = pluginStore;
  }

  authenticate = (credentials: object) => {
    // checks the credentials and returns username if matches.
    return { name: "username" };
  };

  activate(): void {
    this.pluginStore.addFunction(
      "Auth.authenticate",                                   //line 3
      (credentials: object) => this.authenticate(credentials)
    );
  }

  deactivate(): void {
    this.pluginStore.removeFunction("Auth.authenticate");    //line 4
  }
}

export default AuthPlugin;

App.tsx

import React from "react";
import { createPluginStore, PluginProvider } from "react-pluggable";
import AuthPlugin from "./plugins/AuthPlugin";
import Login from "./components/Login";

const pluginStore = createPluginStore();
pluginStore.install(new AuthPlugin());

function App() {
  return (
    <PluginProvider pluginStore={pluginStore}>
      <Login />
    </PluginProvider>
  );
}

export default App;

Login.tsx

import * as React from "react";
import { usePluginStore } from "react-pluggable";

const Login = (props: any) => {
  const pluginStore: any = usePluginStore();

  const submitHandler = () => {
    let credentials = { email: "john@gmail.com", password: "12345" };
    let result = pluginStore.executeFunction("Auth.authenticate", credentials);
    console.log(result);
  };

  return (
    <>
      <h1>Working</h1>
      <button onClick={submitHandler}>Login</button>
    </>
  );
};

export default Login;

Here we are executing the Auth.authenticate method of the Auth plugin, which was installed in App.tsx.

We have seen in the class AuthPlugin that the name of the plugin is used several times. An alternate way is to define a variable that stores the name of the plugin and using that variable in the class anywhere we want the plugin name.

AuthPlugin

import React from "react";
import { IPlugin, PluginStore } from "react-pluggable";

class AuthPlugin implements IPlugin {
  private namespace = "Auth";

  getPluginName(): string {
    return `${this.namespace}@1.0.0`;                                    //line 2
  }

  getDependencies(): string[] {
    return [];
  }

  public pluginStore: any;

  init(pluginStore: PluginStore): void {
    this.pluginStore = pluginStore;
  }

  authenticate = (credentials: object) => {
    // checks the credentials and returns username if matches.
    return { name: "username" };
  };

  activate(): void {
    this.pluginStore.addFunction(
      `${this.namespace}.authenticate`,                                   //line 3
      (credentials: object) => this.authenticate(credentials)
    );
  }

  deactivate(): void {
    this.pluginStore.removeFunction(`${this.namespace}.authenticate`);    //line 4
  }
}

export default AuthPlugin;
