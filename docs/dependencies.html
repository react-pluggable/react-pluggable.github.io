Dependencies

When creating a plugin, getDependencies() is one of the methods we need to implement.

getDependencies(): string[] {
  return []; //plugins
}

It returns an array of plugins in string format which must be installed in your PluginStore before installing this plugin, otherwise, it will throw an error.

PluginStore internally uses semantic versioning for checking whether dependencies are available or not.

We have to provide the full name of the plugin in string format. For instance, if you have a dependency of SharePlugin of version 1.1.0 so in that case you have to return "Share@1.1.0", otherwise it will throw an error.
