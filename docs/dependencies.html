Dependencies

When creating a plugin, getDependencies() is one of the methods we got from IPlugin interface.

getDependencies(): string[] {
  return []; //plugins
}

It returns an array of plugins in string format which must be installed in your PluginStore before installing this plugin, otherwise, it will throw an error.

PluginStore internally uses semantic versioning for checking whether dependencies are available or not.

We have to provide the full name of the plugin in string format. For instance, if you want to return SharePlugin of version 1.1.0 so in that case you have to return "Share@1.1.0" or you can simply write SharePlugin.getPluginName() which return the plugin in string format. While both are correct but the later is preferred conventionally.
